# report-pptx

`reports/{종목명}.md` 분석 리포트를 **고정 양식 PPTX**로 변환하는 스킬입니다.
실제 변환은 `build_pptx.py`가 수행하며, 디자인·순서·가드레일은 코드에 내장돼 있습니다.

## 입출력

- **입력**: `reports/{종목명}.md`
- **출력**: `reports/pptx/{종목명}.pptx` — `pptx` 폴더가 없으면 자동 생성합니다.

## 실행 방법

```bash
python .claude/skills/report-pptx/build_pptx.py "{종목명}" --root . --date 2026-06-16
```

- `--date` 생략 시 오늘 날짜를 작성일로 사용합니다.
- 경로를 직접 지정하려면: `--in reports/삼성전자.md --out reports/pptx/삼성전자.pptx`

## 슬라이드 순서 (항상 고정)

1. **표지** — 종목명 · 작성일 · 작성 주체
2. **종목 개요**
3. **재무 요약** (표 + 최근 3개년)
4. **가격 / 추세**
5. **뉴스 · 심리**
6. **리스크**
7. **한 줄 종합**

## 입력 마크다운 작성 규칙

스킬이 섹션을 인식하려면 `##` 헤더에 아래 키워드를 포함합니다(부분 일치).

| 슬라이드 | 헤더 키워드(아무거나) |
|---|---|
| 종목 개요 | 개요 / 기업개요 |
| 재무 요약 | 재무 / 실적 |
| 가격/추세 | 가격 / 추세 / 차트 / 주가 |
| 뉴스·심리 | 뉴스 / 심리 / 이슈 / 센티 |
| 리스크 | 리스크 / 위험 |
| 한 줄 종합 | 종합 / 결론 / 의견 |

- **표**는 표준 마크다운 파이프 표(`| ... |`)로 작성합니다. 헤더행 다음 `---` 구분선은 자동 무시됩니다.
- 표가 없는 섹션은 `-`/`*` 글머리표 또는 일반 문단을 본문으로 렌더링합니다.
- 매핑되지 않는 헤더(예: 최상단 `# 제목`)는 슬라이드로 만들지 않습니다.

## 디자인 (코드 내장 — 임의 변경 금지)

- 배경 화이트 · 제목 딥네이비 `#1F2A44` · 본문 그레이
- 한화 오렌지 `#F37321` 는 **표지 상단 바 · 제목 하단 구분선 · 강조**에만 절제해 사용
- 표: **헤더행만 딥네이비 배경 + 흰 글자**, 본문행은 얇은 그레이 괘선 (리서치센터 표 양식)
- 한글 폰트 **맑은 고딕 단일 고정** (글자 깨짐 방지)
- 16:9 비율

## 가드레일 (CLAUDE.md 준수)

- **수치 옆 (출처/기준일) 표기** — 입력 .md 작성 시 모든 숫자에 출처를 함께 적습니다. 출처 없는 숫자는 싣지 않습니다.
- **매수/매도·목표가 단정 표현 금지** — 판단 근거까지만. 슬라이드 문구에 직접 권유를 넣지 않습니다.
- **표 잘림 방지** — 표가 슬라이드 영역을 넘으면 코드가 행을 자동으로 줄이고 "일부 생략(원본 .md 참조)" 안내행을 추가합니다. 행이 많으면 입력 .md에서 핵심 행만 남기세요.
- 내용이 비어 있는 섹션은 "확인 불가"로 채워집니다.

## 사용 흐름 (권장)

1. 애널리스트 팀 결과를 `reports/{종목명}.md`에 위 섹션 규칙대로 정리합니다.
2. 위 명령으로 PPTX를 생성합니다.
3. 실행 로그의 "확인 불가/생략" 경고를 확인하고 필요 시 .md를 보완해 재실행합니다.
